# Internationalization Reference — Grabee

## String Resources Location

All strings live in `core:resource` — the only module with Compose Multiplatform resources:

```
core/resource/src/commonMain/composeResources/
├── values/
│   └── strings.xml        ← English (default)
└── values-ja/
    └── strings.xml        ← Japanese
```

The `core:resource` module is configured with `publicResClass = true`, generating `Res` as a public class accessible from all modules.

---

## Accessing Strings in Composables

```kotlin
import org.jetbrains.compose.resources.stringResource
import me.matsumo.grabee.core.resource.Res
import me.matsumo.grabee.core.resource.common_ok
import me.matsumo.grabee.core.resource.error_network

// Simple string
Text(text = stringResource(Res.string.common_ok))

// With format arguments (%1$s, %1$d pattern)
Text(text = stringResource(Res.string.unit_day_before, 3))  // "3 days ago"
```

---

## Accessing Strings in ViewModels (non-composable)

Pass `StringResource` directly — do not call `stringResource()` outside a composable:

```kotlin
// In ViewModel
ScreenState.Error(Res.string.error_network)
ScreenState.Error(
    message = Res.string.error_network,
    retryTitle = Res.string.common_retry,
)
```

The composable layer (`AsyncLoadContents` / `ErrorView`) resolves the `StringResource` to a string.

---

## Adding a New String

1. Add the entry to `values/strings.xml` (English):
   ```xml
   <string name="feature_xyz_title">My New Screen</string>
   ```
2. Add the Japanese translation to `values-ja/strings.xml`:
   ```xml
   <string name="feature_xyz_title">新しい画面</string>
   ```
3. Rebuild — KSP generates the accessor `Res.string.feature_xyz_title` automatically.

---

## String Format Arguments

Grabee uses `%1$s` (string) and `%1$d` (integer) positional format args:

```xml
<string name="unit_day_before">%1$d days ago</string>
<string name="unit_favorite">%1$d likes</string>
```

```kotlin
stringResource(Res.string.unit_day_before, 5)   // "5 days ago"
```

## Font Resources

Custom fonts are in `core/resource/src/commonMain/composeResources/font/`:
- `google_sans_bold.ttf`
- `google_sans_medium.ttf`
- `google_sans_regular.ttf`

Access via `Font(Res.font.google_sans_regular)` in theme typography setup.

## Existing Locale Support

| Locale | Directory |
|---|---|
| English | `values/` (default) |
| Japanese | `values-ja/` |

The system automatically selects the appropriate locale at runtime via Compose Multiplatform's resource system — no manual locale handling needed.
